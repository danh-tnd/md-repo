# **BÁO CÁO ĐIỀU TRA – UPGRADE NODE.JS CHO WEBUI

(Phạm vi: modules/leia-web-frontend)**

## **1) Bối cảnh**

Khi thực hiện local build trên Windows 11, module **`modules/leia-web-frontend`** gặp nhiều lỗi build do:

-   Dự án sử dụng **Node.js cũ**, không còn tương thích với Node 18.
-   Build system vẫn dùng **Gulp 3** và nhiều plugin **không còn tương thích**.
-   Cú pháp và pipeline cũ của Gulp không chạy được trên Node ≥ 16/18.
-   Khi nâng Node, phát sinh nhiều lỗi thực tế như:
    -   ENOENT (thiếu thư mục trong `src/features/…`)
    -   `Can only pipe to one destination`
    -   Thiếu module `merge-stream`
    -   `primordials is not defined`
    -   `Did you forget to signal async completion?`

Do đó cần nâng cấp Node + Gulp + plugin và chỉnh lại toàn bộ build script.

----------

## **2) Nguyên nhân lỗi**

### **(1) Gulp 3 không hoạt động trên Node mới**

-   Gulp 3 phụ thuộc vào API cũ (`graceful-fs`, `orchestrator`).
-   Node ≥ 16 làm phát sinh lỗi `primordials is not defined`.
-   Toàn bộ task không còn chạy được.

### **(2) Plugin Gulp cũ không còn tương thích**

-   `event-stream` → deprecated, không tương thích Node mới
-   `gulp-minify-css`, `gulp-minify-html` → lỗi runtime
-   `gulp-clone` → mất tương thích, gây lỗi pipe
-   Thiếu `merge-stream` khi migrate

### **(3) Cách viết task cũ của Gulp 3 không còn hợp lệ**

-   Gulp 4 yêu cầu **task phải return stream/promise**, pipeline cũ không đáp ứng → sinh lỗi:
    -   `Did you forget to signal async completion?`

### **(4) Sai luồng pipe trong JSStrategy**

-   Lỗi thực tế:  
    `Error: Can only pipe to one destination`
-   Nguyên nhân: dùng cùng một stream pipe sang nhiều đích mà **không clone stream**.

### **(5) JSONUtil cũ gây crash khi file manifest rỗng/mất**

-   Không kiểm tra tồn tại file
-   `JSON.parse` không được bọc try/catch  
    → Dẫn đến build crash.

### **(6) Thiếu thư mục trong dự án**

-   Lỗi thật tế build log:
    -   ENOENT: `src/features/WithdrawFeatures`
    -   ENOENT: `src/features/TenantFeatures`
    -   ENOENT: `src/screens/desktop/error`  
        → buộc phải tạo thủ công để tiếp tục build.

----------

## **3) Nội dung sửa đổi (modules/leia-web-frontend)**

### **(1) Cập nhật cú pháp cũ để phù hợp với Gulp 4**

-   Chuyển toàn bộ `gulp.task("x", ["a","b"])` → `gulp.series()` / `gulp.parallel()`.
-   Viết lại toàn bộ task theo chuẩn:
    -   Dùng `async function` hoặc return stream/promise.
-   Loại bỏ toàn bộ sử dụng callback `done()` không cần thiết.
-   Xây lại pipeline sao cho Gulp 4 kiểm soát đúng tiến trình.

### **(2) Cập nhật plugin**

-   `gulp-minify-css` → `gulp-clean-css`
-   `gulp-minify-html` → `gulp-htmlmin`
-   `event-stream` → `merge-stream`
-   `gulp-clone` → `cloneable-readable`
-   Nâng cấp `gulp-uglify`, `gulp-sourcemaps`, `del`, `vinyl-fs`, `graceful-fs`

### **(3) Cập nhật JSStrategy**

-   Viết lại hoàn toàn theo chuẩn:
    -   baseStream → cloneable → 3 nhánh:
        -   `scripts.js`
        -   `scripts.js.gz`
        -   `scripts.js.manifest`
    -   `return merge(...)` để Gulp 4 đợi hoàn tất
-   Fix lỗi:
    -   `Can only pipe to one destination`
    -   Cài thiếu `merge-stream`

### **(4) Cập nhật CSSStrategy**

-   Áp dụng pattern giống JS:
    -   `gulp-clean-css`
    -   cloneable-readable
    -   3 nhánh output (.css, .gz, .manifest)
-   `return merge(cssFile, gzFile, manifestFile)`

### **(5) Cập nhật HTMLStrategy**

-   Loại `gulp-minify-html`
-   Thêm `gulp-htmlmin` với cấu hình:
    
    htmlmin({  collapseWhitespace:  true  })
    
    `</span><span attribution="{&quot;name&quot;:&quot;Copilot&quot;,&quot;oid&quot;:&quot;E64C3D4F-5E12-4514-AD9B-893A6FAFD00C&quot;,&quot;id&quot;:&quot;E64C3D4F-5E12-4514-AD9B-893A6FAFD00C&quot;,&quot;userInfo&quot;:{&quot;name&quot;:&quot;Copilot&quot;,&quot;oid&quot;:&quot;E64C3D4F-5E12-4514-AD9B-893A6FAFD00C&quot;,&quot;id&quot;:&quot;E64C3D4F-5E12-4514-AD9B-893A6FAFD00C&quot;},&quot;timestamp&quot;:1773740400000,&quot;dataSource&quot;:0}">`
    

### **(6) Cập nhật JSONUtil**

-   Thêm: if  (!fs.existsSync(...))  return  {};
-   Bọc JSON.parse trong try/catch
-   Khi lỗi → log warning, trả `{}`  
    → tránh crash khi thiếu manifest.

### **(7) Cập nhật gulpfile.js**

-   Chuyển toàn bộ sang chuẩn Gulp 4:
    -   `clean` → return del(...)
    -   `cssbuild`, `jsbuild`, `htmlbuild`, `resourcebuild` → return đúng stream
-   Fix lỗi thực tế:
    -   tạo `./target/build` nếu chưa có
    -   tạo manifest rỗng nếu thiếu
    -   thêm delay 500ms để đảm bảo file được generate trước khi gọi HTMLStrategy

### **(8) package.json**

-   Node: "engines":  {  "node":  ">=18 <19"  }
-   Thêm dependency còn thiếu trong log (`merge-stream`, `cloneable-readable`, …)
-   Cập nhật resolution `graceful-fs` để tránh lỗi primordials

### **(9) Sửa các lỗi ENOENT**

-   Tạo các thư mục còn thiếu:
    -   `src/features/WithdrawFeatures`
    -   `src/features/TenantFeatures`
    -   `src/screens/desktop/error`
