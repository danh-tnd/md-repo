## 3. `modules/leia-dmpapp-cs-ui/gulpfile.js`

### Bản chất thay đổi

- Từ Gulp 3 sang **Gulp 4**:
  - Thay `event-stream` → `merge-stream`.
  - Loại bỏ array deps kiểu `gulp.task("x", ["a","b"], fn)` → dùng `gulp.series`.
  - Khai báo function `cleanDist`, `cleanJSBuild`,… rồi dùng `series`.
  - Cuối file tạo các biến:
    ```js
    var js = series(...);
    var jsIFT = series(...);
    var coverage = series(...);
    var css = series(...);
    var html = series(...);
    var htmlIFT = series(...);
    var buildXML = series(...);
    var build = series(buildXML, html);
    var ift = series(buildXML, htmlIFT, packit);
    var defaultTask = series(build, packit);
    ```
    và gán lại `gulp.task("js", js);`…  

### Đánh giá “dư thừa”?

- Đa phần thay đổi là **bắt buộc** để chạy được với Gulp 4; không phải “dọn cho đẹp”.
- Điểm có thể tối giản:
  - Bạn vẫn giữ `var gulp = require("gulp");` (ES5), không đổi sang `const` → đúng với tiêu chí “thay ít nhất có thể”. Không vấn đề.
  - Bạn thêm alias `var series = gulp.series;` → hợp lý để code gọn; không dư.
  - `packit` giờ trả `merge(...)` thay vì `es.concat(...)` → bắt buộc vì bỏ `event-stream`.
- Không thấy logic mới “quá đà” hoặc code comment thừa trong file này.

→ **Giữ nguyên là hợp lý.** Không có thay đổi nào rõ ràng là không cần thiết.

## 4. `modules/leia-dmpapp-cs-ui/gulpscript/*.js`

### CSSStrategy

- `gulp-minify-css` → `gulp-clean-css`, `event-stream` → `merge-stream`.  
- Thay đúng tại chỗ, không thêm cấu trúc mới.
- Không dư.

### HTMLStrategy

- `event-stream` → `merge-stream`.
- `gulp-minify-html` → `gulp-htmlmin` (với options collapseWhitespace/removeComments/keepClosingSlash).
- Chỗ dễ dư:
  ```js
  return merge.apply(null, streams);
  ```
  và rồi ở ngoài:
  ```js
  var htmlstream = createHtmlStream().pipe(clone())
      .pipe(gulp.dest(opts.dest));

  var gzstream = createHtmlStream()
      .pipe(gzip())
      .pipe(gulp.dest(opts.dest));

  var manifeststream = createHtmlStream()
      .pipe(rev())
      .pipe(rev.manifest())
      .pipe(gulp.dest(opts.dest));
  ```
  - Gọi `createHtmlStream()` **ba lần** thay vì dùng clone → hơi lặp, nhưng đây là “logic mới” bạn thêm; dù vậy nó thay thế pattern cũ dùng `clone` + `es.merge`.
  - Nếu muốn “ít thay đổi nhất”:
    - Có thể giữ pattern cũ: 1 stream base, clone ra 3 nhánh, rồi `merge` thay vì `es.merge`.  
    - Nhưng hiện tại cách bạn làm vẫn đúng chức năng.  
  → Đây là **refactor khá sâu**; nếu muốn tối thiểu, có thể quay lại pattern gần với code cũ hơn (single base stream + clone + merge) thay vì re-build stream 3 lần.  

### JSStrategy

- Thay `jsStringsTask` từ việc trả về mảng stream → callback `cb()` và viết file bằng fs (`_buildStringFile` mới).
- Đây là **thay đổi lớn về cách build chuỗi**, không đơn thuần là Gulp 4 migration.
- Nếu mục tiêu chỉ là Gulp 4 + Node 18:
  - Có thể vẫn dùng pipeline cũ với `merge-stream` thay `event-stream`, và tránh viết trực tiếp file bằng `fs`.
  - Việc chuyển sang `fs.writeFileSync` + hash MD5 là một enhancement thêm; không bắt buộc cho việc nâng Node/Gulp.

→ Nếu muốn “ít thay đổi nhất”, phần `_buildStringFile`/`jsStringsTask` là nơi **đang thay đổi nhiều hơn cần thiết**.

## 7. `modules/leia-web-frontend/buildlib/CSSStrategy.js`

- `gulp-minify-css` → `gulp-clean-css`, `event-stream` → `merge-stream`, `gulp-clone` → `cloneable-readable`.
- Bạn thêm `createBaseStream`, rồi dùng:
  ```js
  var base = cloneable(createBaseStream(srcs, opts));

  var cssFile = base.clone().pipe(gulp.dest(...));
  var gzFile = base.clone().pipe(gzip()).pipe(gulp.dest(...));
  var manifestFile = base.clone().pipe(rev.manifest(...)).pipe(gulp.dest(...));

  return merge(cssFile, gzFile, manifestFile);
  ```
- Đây là **một refactor sạch** và tương đối tối giản; rất phù hợp với Gulp 4 và multi-dest pattern.
- Các comment tiếng Việt (“CHỈ CẦN RETURN...”) là “noise” nhẹ; nếu muốn code tối giản & production-like, có thể **xóa comment hướng dẫn** để diff ngắn hơn.

Không có stream thừa; tất cả branch đều được merge.

## 9. `modules/leia-web-frontend/buildlib/JSONUtil.js`

Bạn từ:

```js
Params.require(src);
data = readFile(src);
return JSON.parse(data);
```

sang:

- Check `fs.existsSync(src)` → `{}` nếu không tồn tại.
- Nếu file rỗng → `{}`.
- Catch parse error, log warning, return `{}`.

Đây là **thay đổi hành vi**: trước đây lỗi JSON sẽ làm build fail; giờ build tiếp tục. Thay đổi này giúp bạn tránh crash do manifest trống/mất.

Về “ít thay đổi nhất”:
- Về mặt LOC, đã tăng khá nhiều, nhưng chức năng rất rõ và cần thiết để chống ENOENT/JSON parse error.
- Nếu muốn minimal hơn, bạn có thể:
  - Chỉ bọc JSON.parse bằng try/catch mà không thêm exists/trim check. Nhưng như vậy build vẫn fail nếu thiếu file.
- Với tình trạng manifest đôi khi chưa được flush xong, logic an toàn này rất hữu ích. Mình không coi là “dư thừa”.

## 10. `modules/leia-web-frontend/buildlib/JSStrategy.js`

- Loại bỏ `event-stream`, `gulp-clone`; thêm `cloneable-readable` + `merge-stream`.
- `jsTask` được viết lại theo pattern:
  - base stream → concat/uglify/rev → optional sourcemaps → wrap bằng `cloneable` → 3 nhánh (.js, .gz, .manifest) → `merge`.
- Đây là refactor sâu hơn bản cũ (vốn tạo `jsstream`, rồi `clone()` 2 lần và `es.merge`), nhưng:
  - Về mặt Gulp 4, pattern mới là “đúng chuẩn”.
  - Không thấy stream thừa: cả 3 branch đều dùng trong merge; không có biến nào unused.

Giống CSSStrategy, comment tiếng Việt trong code (“ví dụ: scripts.js.manifest”, “Gộp tất cả nhánh...”) là **optional**; có thể xóa để code gọn, diff ngắn.

## 11. `modules/leia-web-frontend/gulpfile.js`

Đây là file bạn vừa show bản hiện tại.

Thay đổi chính:

- `var` → `const` cho require + `ARGUMENTS_START`.
- Thêm `del`, `fs`, `path`.
- Gulp 3 → Gulp 4:
  - Bỏ `cleanTask("./target")`, `["dep1", "dep2"]` style.
  - Thêm task `clean` dùng `del(["./target/build"])`.
  - `messagefilebuild`, `cssbuild`, `jsbuild`, `htmlbuild`, `resourcebuild`, `package`, `build`, `default` đều chuyển thành function/`gulp.series` với return stream/promise chuẩn.

Về mức độ thay đổi:

- **Essential**:
  - Chuyển sang `gulp.series` và đảm bảo mỗi task return stream/promise → bắt buộc.
  - Thay logic async của `package`, `htmlbuild`, `jsbuild`, `cssbuild` → tránh “Did you forget to signal async completion?”.
- **Có thể xem là dư một chút**:
  - Đổi tất cả `var` sang `const` trong file: đây là “style modernisation”.
    - Nếu tiêu chí “ít thay đổi nhất” rất nghiêm, có thể giữ `var`; nhưng Node 18 + modern code thì `const` cũng rất hợp lý.
  - Comment tiếng Việt mô tả cách Gulp hoạt động (ví dụ: `// Gọi Strategy... BẮT BUỘC phải return stream/promise`, `// ĐỪNG QUÊN cặp ngoặc...`):
    - Chỉ hữu ích trong giai đoạn debug; với code production, có thể xóa bớt để diff nhỏ và code sạch.

## Tóm tắt: chỗ nào đang “nhiều hơn tối thiểu”?

Nếu bạn muốn cực kỳ tối giản so với yêu cầu “nâng Node 16/18 và Gulp 4, fix build” thì:

1. **DMPApp CS UI – JSStrategy.js**
   - Thay đổi `_buildStringFile` + `jsStringsTask` là nhiều nhất, vượt quá việc “chỉ đổi event-stream → merge-stream”.
   - Nếu không có bug cụ thể buộc phải thay, đây là nơi đáng cân nhắc quay lại pattern cũ với `merge-stream` để diff gọn hơn.

2. **Test `leia_spec.js`**
   - Mock `$` + `setupAppForLeiaSpec` khá chi tiết;
   - Dù hợp lý, nhưng nếu trước đây test đã chạy được, đây là vùng “refactor/ cải tiến” chứ không chỉ upgrade.

3. **Comment giải thích bằng tiếng Việt trong build scripts**
   - Các comment kiểu:
     - `// CHỈ CẦN RETURN...`
     - `// ĐỪNG QUÊN cặp ngoặc thực thi này`
     - `// Gulp 4: ...`
   - Có thể xóa để code sạch & diff ngắn hơn, không ảnh hưởng logic.

4. **package.json – dependencies không còn dùng**
   - `gulp-minify-css`, `gulp-minify-html`, `event-stream` (nếu thực sự không còn require) hiện là “thừa”.
   - Tuy nhiên, gỡ chúng ra cũng là thêm diff; nếu bạn ưu tiên “ít thay đổi logic” hơn “sạch dependency”, có thể giữ lại tạm.

Ngoài các điểm trên, đa số thay đổi còn lại là **cần thiết** cho việc:

- Chạy được với Node 18,
- Nâng Gulp lên 4,
- Loại bỏ plugin cũ/deprecated,
- Sửa các vấn đề async/stream.

Nếu bạn muốn, mình có thể đề xuất một patch cụ thể để:
- Xóa bớt comment “giải thích”,
- (Tuỳ chọn) trả lại `jsStringsTask` của CS UI về pattern gần với cũ nhưng dùng `merge-stream`,
- (Tuỳ chọn) đánh dấu các devDependency thực sự không còn require để bạn quyết định uninstall sau.
